# BambuLab Filament Tag Generator

Generate 3D-printable filament tags for BambuLab spools. Each tag has a white base and black frame/text, exported as a multi-color 3MF that BambuStudio recognizes out of the box.

## How it works

```
scraper.py              generator.py
 (scrape BambuLab)  -->   (OpenSCAD + 3MF packing)
       |                        |
   CSV file                colored 3MF files
   (filament data)     (ready to print in BambuStudio)
```

1. **Scrape** filament data from the BambuLab website into a CSV
2. **Generate** 3MF tags from the CSV, with white base + black text/frame already color-assigned

## Requirements

- Python 3.10+
- [OpenSCAD](https://openscad.org/downloads.html) (must be in your PATH)
- The [Overpass](https://fonts.google.com/specimen/Overpass) font installed (used for tag text)
- Python packages for the scraper only:

```bash
pip install requests beautifulsoup4
```

pandas + openpyxl are optional (only needed for Excel export).

The tag generator itself has **no Python dependencies** beyond the standard library.

## Printing

Each filament label consists of two parts that snap together:

1. **Spool clip** (`templates/spool_clip.3mf`) — print once per spool, clips onto the spool hole
2. **Name tag** — generated by this tool, slides into the clip

Print the spool clip first, then generate and print the name tags for your filaments.

## Quick start

```bash
# 1. Scrape all filament data from BambuLab store
python3 scraper.py --all

# 2. Generate a plate of specific tags by color code
python3 generator.py --codes 10100,10201,33102 --printer h2d

# 3. Open the 3MF in BambuStudio and print
open output/custom_plate.3mf
```

## Tag generator usage

### Pick specific tags by color code

```bash
# Generate a single plate with specific filament codes
python3 generator.py --codes 10100,10201,33102

# With printer bed size
python3 generator.py --codes 10100,10201,33102 --printer h2d

# Mix codes from any collection (PLA + PETG + TPU, etc.)
python3 generator.py --codes 10100,33102,50200,11100
```

Output: `output/custom_plate.3mf`

Unknown codes are warned and skipped.

### Generate plates by category

```bash
# One 3MF per category with all tags arranged on a grid
python3 generator.py --plate

# Specify printer for correct bed size
python3 generator.py --plate --printer a1mini
python3 generator.py --plate --printer h2d
```

Output: `output/plates/PLA_Basic.3mf`, `PLA_Matte.3mf`, etc.

Auto-splits into multiple plates if too many tags for one bed.

### Generate individual tags

```bash
# One 3MF per tag, organized by collection
python3 generator.py
```

Output: `output/PLA/PLA_Basic_Jade_White_10100.3mf`, etc.

### Other options

```bash
# Use a specific CSV file
python3 generator.py --csv path/to/filaments.csv

# Limit to N tags (useful for testing)
python3 generator.py --limit 5

# Verbose output
python3 generator.py --codes 10100,10201 -v
```

## Scraper usage

The scraper pulls filament names, color codes, and categories from the BambuLab store.

```bash
# Auto-discover and scrape ALL filaments (global store, default)
python3 scraper.py --all

# Use a specific regional store
python3 scraper.py --all --store us     # United States
python3 scraper.py --all --store eu     # Europe
python3 scraper.py --all --store uk     # United Kingdom
python3 scraper.py --all --store au     # Australia
python3 scraper.py --all --store ca     # Canada
python3 scraper.py --all --store asia   # Asia
python3 scraper.py --all --store kr     # South Korea
python3 scraper.py --all --store jp     # Japan

# If some pages get rate-limited, retry just the failed ones
# (the script saves failed_urls.txt and tells you the command)
python3 scraper.py --url-file failed_urls.txt --merge

# Scrape specific URLs
python3 scraper.py --urls https://store.bambulab.com/en/products/pla-basic-filament
```

### Output files

| File | Description |
|------|-------------|
| `filament_data_combined.csv` | All filaments in one file (used by generator) |
| `PLA.csv`, `PETG.csv`, etc. | One CSV per collection |
| `filament_data.xlsx` | Excel with one sheet per collection (requires pandas) |

## How the 3MF color trick works

BambuStudio ignores the standard 3MF `<basematerials>` element. It only reads `<m:colorgroup>` / `<m:color>` from the [3MF Materials Extension](http://schemas.microsoft.com/3dmanufacturing/material/2015/02). The generator builds the 3MF XML directly with this format, so BambuStudio auto-assigns:

- **Extruder 1**: white base (`#FFFFFFFF`)
- **Extruder 2**: black frame + text (`#000000FF`)

No manual color assignment needed after import.
